# 모델 설정 (탐색)

이 문서는 향후 모델 설정에 대한 **아이디어**를 정리한 것입니다. 배포되는 사양이 아닙니다. 현재 동작에 대해서는 다음을 참조하세요:

- [모델](/concepts/models)
- [모델 폴백](/concepts/model-failover)
- [OAuth + 프로필](/concepts/oauth)

## 배경

운영자는 다음을 원합니다:

- 프로바이더당 여러 인증 프로필 (개인용 vs 업무용).
- 예측 가능한 폴백이 있는 간단한 `/model` 선택.
- 텍스트 모델과 이미지 가능 모델 간의 명확한 구분.

## 가능한 방향 (높은 수준)

- 모델 선택을 간단하게 유지합니다: `provider/model` (선택적 별칭 포함).
- 프로바이더가 명시적 순서가 있는 여러 인증 프로필을 가질 수 있도록 합니다.
- 모든 세션이 일관되게 폴백되도록 전역 폴백 목록을 사용합니다.
- 명시적으로 구성된 경우에만 이미지 라우팅을 재정의합니다.

## 미해결 질문

- 프로필 회전이 프로바이더당인지 모델당인지?
- UI에서 세션에 대한 프로필 선택을 어떻게 표시해야 하는가?
- 레거시 설정 키에서 가장 안전한 마이그레이션 경로는 무엇인가?
