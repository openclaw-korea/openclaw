# 플러그인 매니페스트 (openclaw.plugin.json)

모든 플러그인은 **플러그인 루트**에 `openclaw.plugin.json` 파일을 포함해야 합니다.
OpenClaw는 이 매니페스트를 사용하여 플러그인 코드를 실행하지 않고도 설정을 유효성 검사합니다. 누락되거나 유효하지 않은 매니페스트는 플러그인 오류로 취급되며 설정 유효성 검사를 차단합니다.

전체 플러그인 시스템 가이드를 참조하세요: [플러그인](/plugin).

## 필수 필드

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

필수 키:

- `id` (문자열): 정규 플러그인 ID입니다.
- `configSchema` (객체): 플러그인 설정용 JSON 스키마입니다 (인라인).

선택적 키:

- `kind` (문자열): 플러그인 종류입니다 (예: `"memory"`).
- `channels` (배열): 이 플러그인이 등록하는 채널 ID입니다 (예: `["matrix"]`).
- `providers` (배열): 이 플러그인이 등록하는 프로바이더 ID입니다.
- `skills` (배열): 로드할 스킬 디렉토리입니다 (플러그인 루트 기준 상대 경로).
- `name` (문자열): 플러그인의 표시 이름입니다.
- `description` (문자열): 플러그인의 간단한 설명입니다.
- `uiHints` (객체): UI 렌더링을 위한 설정 필드 레이블/플레이스홀더/민감도 플래그입니다.
- `version` (문자열): 플러그인 버전입니다 (정보 제공용).

## JSON 스키마 요구 사항

- **모든 플러그인은 설정을 허용하지 않는 경우에도 JSON 스키마를 포함해야 합니다**.
- 빈 스키마는 허용됩니다 (예: `{ "type": "object", "additionalProperties": false }`).
- 스키마는 런타임이 아닌 설정 읽기/쓰기 시점에 유효성 검사됩니다.

## 유효성 검사 동작

- 알 수 없는 `channels.*` 키는 채널 ID가 플러그인 매니페스트에 선언되지 않은 한 **오류**입니다.
- `plugins.entries.<id>`, `plugins.allow`, `plugins.deny`, `plugins.slots.*`는 **발견 가능한** 플러그인 ID를 참조해야 합니다. 알 수 없는 ID는 **오류**입니다.
- 플러그인이 설치되었으나 매니페스트나 스키마가 손상되었거나 누락된 경우 유효성 검사는 실패하고 Doctor는 플러그인 오류를 보고합니다.
- 플러그인 설정이 존재하지만 플러그인이 **비활성화**된 경우, 설정은 유지되고 Doctor 및 로그에 **경고**가 표시됩니다.

## 참고 사항

- 매니페스트는 로컬 파일 시스템 로드를 포함한 **모든 플러그인에 필수**입니다.
- 런타임은 여전히 플러그인 모듈을 별도로 로드합니다. 매니페스트는 디스커버리 + 유효성 검사용입니다.
- 플러그인이 네이티브 모듈에 의존하는 경우, 빌드 단계와 패키지 관리자 허용 목록 요구 사항을 문서화하세요 (예: pnpm `allow-build-scripts` - `pnpm rebuild <package>`).
