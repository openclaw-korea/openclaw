# 음성 웨이크 (전역 웨이크 워드)

OpenClaw는 **웨이크 워드를 게이트웨이가 소유하는 단일 전역 목록**으로 처리합니다.

- **노드별 사용자 지정 웨이크 워드는 없습니다**.
- **모든 노드/앱 UI가 편집**할 수 있습니다. 변경 사항은 게이트웨이에 의해 유지되고 모두에게 브로드캐스트됩니다.
- 각 디바이스는 여전히 자체 **음성 웨이크 활성화/비활성화** 토글을 유지합니다(로컬 UX + 권한이 다름).

## 저장소 (게이트웨이 호스트)

웨이크 워드는 게이트웨이 머신에 다음 위치에 저장됩니다:

- `~/.openclaw/settings/voicewake.json`

형식:

```json
{ "triggers": ["openclaw", "claude", "computer"], "updatedAtMs": 1730000000000 }
```

## 프로토콜

### 메서드

- `voicewake.get` → `{ triggers: string[] }`
- `voicewake.set` 매개변수 `{ triggers: string[] }` → `{ triggers: string[] }`

참고사항:

- 트리거는 정규화됩니다(공백 제거, 빈 것 삭제). 빈 목록은 기본값으로 대체됩니다.
- 안전을 위해 제한이 시행됩니다(개수/길이 제한).

### 이벤트

- `voicewake.changed` 페이로드 `{ triggers: string[] }`

누가 받는지:

- 모든 WebSocket 클라이언트(macOS 앱, WebChat 등).
- 모든 연결된 노드(iOS/Android), 그리고 노드 연결 시 초기 "현재 상태" 푸시로도 받습니다.

## 클라이언트 동작

### macOS 앱

- `VoiceWakeRuntime` 트리거를 제어하기 위해 전역 목록을 사용합니다.
- 음성 웨이크 설정에서 "트리거 워드" 편집은 `voicewake.set`을 호출한 다음 브로드캐스트에 의존하여 다른 클라이언트를 동기화 상태로 유지합니다.

### iOS 노드

- `VoiceWakeManager` 트리거 감지를 위해 전역 목록을 사용합니다.
- 설정에서 웨이크 워드 편집은 게이트웨이 WS를 통해 `voicewake.set`을 호출하고 로컬 웨이크 워드 감지를 반응적으로 유지합니다.

### Android 노드

- 설정에서 웨이크 워드 편집기를 노출합니다.
- 게이트웨이 WS를 통해 `voicewake.set`을 호출하여 편집 사항이 모든 곳에서 동기화되도록 합니다.
