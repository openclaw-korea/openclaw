# 온보딩 (macOS 앱)

이 문서는 **현재** 첫 실행 온보딩 흐름을 설명합니다. 목표는 부드러운 "첫날" 경험입니다: 게이트웨이 실행 위치 선택, 인증 연결, 마법사 실행, 에이전트가 스스로 부트스트랩하게 하기.

## 페이지 순서 (현재)

1. 환영 + 보안 안내
2. **게이트웨이 선택** (로컬 / 원격 / 나중에 설정)
3. **인증 (Anthropic OAuth)** — 로컬 전용
4. **설정 마법사** (게이트웨이 기반)
5. **권한** (TCC 프롬프트)
6. **CLI** (선택사항)
7. **온보딩 채팅** (전용 세션)
8. 준비 완료

## 1) 환영 + 보안 안내

표시된 보안 안내를 읽고 그에 따라 결정하세요.

## 2) 로컬 vs 원격

**게이트웨이**는 어디서 실행됩니까?

- **로컬 (이 Mac):** 온보딩이 OAuth 흐름을 실행하고 자격 증명을 로컬에 작성할 수 있습니다.
- **원격 (SSH/Tailnet을 통해):** 온보딩이 로컬에서 OAuth를 실행하지 **않습니다**; 자격 증명은 게이트웨이 호스트에 있어야 합니다.
- **나중에 설정:** 설정을 건너뛰고 앱을 구성되지 않은 상태로 둡니다.

게이트웨이 인증 팁:

- 마법사는 이제 루프백에서도 **토큰**을 생성하므로, 로컬 WS 클라이언트가 인증해야 합니다.
- 인증을 비활성화하면 모든 로컬 프로세스가 연결할 수 있습니다; 완전히 신뢰할 수 있는 머신에서만 사용하세요.
- 다중 머신 접근이나 비루프백 바인드에는 **토큰**을 사용하세요.

## 3) 로컬 전용 인증 (Anthropic OAuth)

macOS 앱은 Anthropic OAuth (Claude Pro/Max)를 지원합니다. 흐름:

- OAuth용 브라우저 열기 (PKCE)
- 사용자에게 `code#state` 값 붙여넣기 요청
- `~/.openclaw/credentials/oauth.json`에 자격 증명 작성

다른 프로바이더 (OpenAI, 커스텀 API)는 현재 환경 변수나 설정 파일로 구성됩니다.

## 4) 설정 마법사 (게이트웨이 기반)

앱은 CLI와 동일한 설정 마법사를 실행할 수 있습니다. 이를 통해 온보딩이 게이트웨이 측 동작과 동기화되고 SwiftUI에서 로직을 중복하지 않습니다.

## 5) 권한

온보딩은 다음에 필요한 TCC 권한을 요청합니다:

- 알림
- 손쉬운 사용
- 화면 녹화
- 마이크 / 음성 인식
- 자동화 (AppleScript)

## 6) CLI (선택사항)

앱은 npm/pnpm을 통해 전역 `openclaw` CLI를 설치할 수 있어 터미널 워크플로우와 launchd 작업이 바로 작동합니다.

## 7) 온보딩 채팅 (전용 세션)

설정 후 앱은 전용 온보딩 채팅 세션을 열어 에이전트가 자신을 소개하고 다음 단계를 안내할 수 있습니다. 이를 통해 첫 실행 안내가 일반 대화와 분리됩니다.

## 에이전트 부트스트랩 리추얼

첫 에이전트 실행 시 OpenClaw는 워크스페이스를 부트스트랩합니다 (기본값 `~/.openclaw/workspace`):

- `AGENTS.md`, `BOOTSTRAP.md`, `IDENTITY.md`, `USER.md` 시드
- 짧은 Q&A 리추얼 실행 (한 번에 하나의 질문)
- ID + 선호도를 `IDENTITY.md`, `USER.md`, `SOUL.md`에 작성
- 완료되면 `BOOTSTRAP.md`를 제거하여 한 번만 실행되도록 함

## 선택사항: Gmail 훅 (수동)

Gmail Pub/Sub 설정은 현재 수동 단계입니다. 사용:

```bash
openclaw webhooks gmail setup --account you@gmail.com
```

자세한 내용은 [/automation/gmail-pubsub](/ko-KR/automation/gmail-pubsub)을 참조하세요.

## 원격 모드 참고

게이트웨이가 다른 머신에서 실행될 때 자격 증명과 워크스페이스 파일은 **해당 호스트**에 있습니다. 원격 모드에서 OAuth가 필요하면 게이트웨이 호스트에 다음을 생성하세요:

- `~/.openclaw/credentials/oauth.json`
- `~/.openclaw/agents/<agentId>/agent/auth-profiles.json`
