# OpenProse

OpenProse는 AI 세션을 조율하기 위한 이식 가능한 마크다운 중심의 워크플로우 형식입니다. OpenClaw에서는 플러그인으로 제공되며 OpenProse 스킬 팩과 `/prose` 슬래시 명령을 설치합니다. 프로그램은 `.prose` 파일에 있으며 명시적인 제어 흐름을 사용하여 여러 개의 하위 에이전트를 생성할 수 있습니다.

공식 사이트: https://www.prose.md

## 기능

- 명시적 병렬 처리를 사용한 다중 에이전트 연구 및 종합.
- 반복 가능한 승인 안전 워크플로우(코드 검토, 사건 분류, 콘텐츠 파이프라인).
- 지원되는 에이전트 런타임 전체에서 실행할 수 있는 재사용 가능한 `.prose` 프로그램.

## 설치 및 활성화

번들 플러그인은 기본적으로 비활성화되어 있습니다. OpenProse를 활성화하세요:

```bash
openclaw plugins enable open-prose
```

플러그인을 활성화한 후 게이트웨이를 다시 시작하세요.

개발/로컬 체크아웃: `openclaw plugins install ./extensions/open-prose`

관련 문서: [플러그인](/plugin), [플러그인 매니페스트](/plugins/manifest), [스킬](/tools/skills).

## 슬래시 명령

OpenProse는 `/prose`를 사용자가 호출 가능한 스킬 명령으로 등록합니다. OpenProse VM 지침으로 라우팅되며 백그라운드에서 OpenClaw 도구를 사용합니다.

일반적인 명령:

```
/prose help
/prose run <file.prose>
/prose run <handle/slug>
/prose run <https://example.com/file.prose>
/prose compile <file.prose>
/prose examples
/prose update
```

## 예제: 간단한 `.prose` 파일

```prose
# 두 에이전트가 병렬로 실행되는 연구 및 종합

input topic: "What should we research?"

agent researcher:
  model: sonnet
  prompt: "You research thoroughly and cite sources."

agent writer:
  model: opus
  prompt: "You write a concise summary."

parallel:
  findings = session: researcher
    prompt: "Research {topic}."
  draft = session: writer
    prompt: "Summarize {topic}."

session "Merge the findings + draft into a final answer."
context: { findings, draft }
```

## 파일 위치

OpenProse는 작업 공간의 `.prose/` 아래에 상태를 유지합니다:

```
.prose/
├── .env
├── runs/
│   └── {YYYYMMDD}-{HHMMSS}-{random}/
│       ├── program.prose
│       ├── state.md
│       ├── bindings/
│       └── agents/
└── agents/
```

사용자 수준의 지속형 에이전트는 다음 위치에 있습니다:

```
~/.prose/agents/
```

## 상태 모드

OpenProse는 여러 상태 백엔드를 지원합니다:

- **filesystem** (기본값): `.prose/runs/...`
- **in-context**: 소규모 프로그램용 일시적
- **sqlite** (실험적): `sqlite3` 바이너리 필요
- **postgres** (실험적): `psql` 및 연결 문자열 필요

참고:

- sqlite/postgres는 선택적이고 실험적입니다.
- postgres 자격 증명은 하위 에이전트 로그로 흐릅니다. 전용의 최소 권한 DB를 사용하세요.

## 원격 프로그램

`/prose run <handle/slug>`는 `https://p.prose.md/<handle>/<slug>`로 확인됩니다.
직접 URL은 그대로 가져옵니다. 이는 `web_fetch` 도구(또는 POST의 경우 `exec`)를 사용합니다.

## OpenClaw 런타임 매핑

OpenProse 프로그램은 OpenClaw 기본 요소에 매핑됩니다:

| OpenProse 개념         | OpenClaw 도구    |
| ---------------------- | --------------- |
| 세션 생성 / 작업 도구  | `sessions_spawn` |
| 파일 읽기/쓰기        | `read` / `write` |
| 웹 가져오기           | `web_fetch`      |

도구 허용 목록이 이 도구들을 차단하면 OpenProse 프로그램이 실패합니다. [스킬 설정](/tools/skills-config)을 참조하세요.

## 보안 및 승인

`.prose` 파일을 코드처럼 취급하세요. 실행 전에 검토하세요. OpenClaw 도구 허용 목록 및 승인 게이트를 사용하여 부작용을 제어하세요.

결정론적이고 승인이 필요한 워크플로우의 경우 [Lobster](/tools/lobster)와 비교하세요.
